# رویایی رسا — نسخه‌ی کاملاً استاتیک (GitHub Pages)

این ریپو نسخه‌ی **کاملاً استاتیک** سایت «رویایی رسا» است؛ مخصوص انتشار روی
گیت‌هاب و پروفایل/رزومه، بدون نیاز به هیچ سرور PHP یا دیتابیس MySQL.

نسخه‌ی اصلی پروژه یک اپلیکیشن کامل PHP + MySQL با پنل مدیریت است. در این
نسخه، همان رابط کاربری و همان جریان‌های کاربری (لیست/جزئیات مقاله و پروژه،
کامنت، فرم تماس، پنل مدیریت) حفظ شده، اما بک‌اند واقعی با یک لایه‌ی
جاوااسکریپت جایگزین شده که داده‌ها را از یک فایل JSON می‌خواند.

## ساختار

```
index.html, articles.html, article.html, projects.html, project.html,
about.html, author.html, contact.html, search.html   → صفحات عمومی سایت
admin/                                                → پنل مدیریت دمو
data/db.json                                          → «دیتابیس» استاتیک (۱۱ مقاله، ۷ پروژه، نظرات، پیام‌ها و ...)
js/data-shim.js                                       → جایگزین بک‌اند PHP برای صفحات عمومی
admin/assets/admin-static.js                          → منطق پنل مدیریت دمو
```

## چطور کار می‌کند؟

هر جا که کد اصلی سایت با `fetch("api/xxx.php?...")` به سرور درخواست می‌زد،
`js/data-shim.js` این درخواست را می‌گیرد و بدون هیچ شبکه‌ای، از روی
`data/db.json` همان ساختار JSON قبلی را برمی‌گرداند (لیست/جست‌وجو/صفحه‌بندی
مقالات و پروژه‌ها، مقاله/پروژه تکی، نویسنده، تگ‌ها و دسته‌ها، کامنت‌ها،
ثبت کامنت و پیام تماس). یعنی خودِ HTML/CSS/JS اصلی سایت تقریباً بدون تغییر
مانده و فقط منبع داده عوض شده است.

- **کامنت جدید / پیام تماس جدید**: در `localStorage` همان مرورگر ذخیره می‌شود؛
  فقط برای همان کاربر روی همان دستگاه باقی می‌ماند (چون سروری برای ذخیره‌ی
  دائمی وجود ندارد).
- **کپچا**: به‌جای تصویر تولیدشده روی سرور، یک SVG ساده در همان مرورگر تولید
  و اعتبارسنجی می‌شود.
- **شمارنده‌ی بازدید**: برای هر مقاله/پروژه فقط در نشست همان کاربر افزایش
  پیدا می‌کند و به سروری ارسال نمی‌شود.
- **تصاویر**: کاور مقالات و پروژه‌ها از [Picsum Photos](https://picsum.photos)
  (عکس‌های واقعی و لایسنس‌آزاد، seed-based یعنی برای هر آیتم همیشه همان عکس
  ثابت لود می‌شود) و آواتار نویسنده‌ها از [DiceBear](https://dicebear.com)
  (آواتار تصویرسازی‌شده، نه عکس افراد واقعی) گرفته می‌شوند. هر دو سرویس رایگان
  و بدون نیاز به کلید API هستند؛ فقط باید هنگام بازدید از سایت به اینترنت
  وصل باشید (روی GitHub Pages طبیعتاً همینطور است).

## پنل مدیریت (`admin/index.html`)

یک پنل مدیریت کاملاً دمو و سمت‌کاربر، برای نمایش قابلیت طراحی و پیاده‌سازی
پنل ادمین در رزومه:

- ورود با نام کاربری `admin` و رمز `demo1234` (روی خود صفحه‌ی ورود هم نوشته شده)
- داشبورد آماری (تعداد مقالات/پروژه‌ها، نظرات در انتظار تایید، پیام‌های
  خوانده‌نشده، نمودار ساده‌ی بازدید هفتگی)
- مدیریت مقالات و پروژه‌ها (شامل یک مقاله و یک پروژه‌ی پیش‌نویس نمونه):
  ویرایش عنوان/خلاصه/وضعیت، حذف
- صف بررسی نظرات (تایید / رد / حذف) با چند نظر نمونه‌ی در انتظار تایید
- مدیریت پیام‌های فرم تماس (خوانده/نخوانده، حذف)
- تنظیمات (تایید خودکار نظرات، آمار بازدید، بازنشانی کامل داده‌های دمو)

همه‌ی این تغییرات فقط در `localStorage` مرورگر شما ذخیره می‌شوند (کلید
جداگانه از داده‌های سایت عمومی) و با دکمه‌ی «بازنشانی داده‌های دمو» در تب
تنظیمات، به حالت اول برمی‌گردند. **این پنل هیچ داده‌ای را به هیچ سروری
ارسال نمی‌کند** — کاملاً برای نمایش رابط کاربری و جریان کار طراحی شده است.

## انتشار روی GitHub Pages

1. محتوای این پوشه را در ریشه‌ی ریپوی گیت‌هاب قرار دهید (یا در شاخه‌ی `gh-pages`).
2. از تنظیمات ریپو، گزینه‌ی GitHub Pages را روی همان شاخه/پوشه فعال کنید.
3. چون تمام فونت‌ها، Three.js و GSAP به‌صورت لوکال در `assets/vendor/` قرار
   دارند، سایت بدون نیاز به CDN خارجی هم به‌درستی بالا می‌آید.

## محدودیت‌های شناخته‌شده‌ی این نسخه‌ی دمو

- فرم تماس و ثبت کامنت واقعاً به کسی ایمیل/پیام ارسال نمی‌کنند.
- تغییرات پنل مدیریت بین کاربران/مرورگرهای مختلف مشترک نیست.
- برای نسخه‌ی کامل با بک‌اند PHP/MySQL واقعی (احراز هویت واقعی، ارسال
  ایمیل، ذخیره‌ی دائمی)، باید همان نسخه‌ی اصلی پروژه روی یک هاست PHP اجرا شود.
